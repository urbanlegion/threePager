Q: Which is a better syntax for if condition, and why?

a. if ("John" == $name)

b. if ($name == "John")

Answer

Let's start by saying that it 100% works, yet I would have to say that B is the correct answer, not only because it's considered best practice but i'm just assuming that the compiler plays some part here when it's looking for a var that contains the string "John", while in B, the compiler already has the var to check if it contains the string "John", that's my unchecked assumption.

Moreover, language as in any language, has certain syntax boundaries, not only for compilers but for other human coders as well. While no coder has ever asked "if the chocolate is cake", many coders do ask "if the cake is chocolate", common practice.

Q: What would be the output of the following class

class student{
    protected $name;
    protected $lastName;

    public function setName($name){
        $this->name = $name;
    }

    public function setLastName($lastName){
        $this->lastName = $lastName;
    }

    public static function getName(){
        return $this->name;
    }

    public static function getLastName(){
        return $this->lastName;
    }
}

And the following script:

$student = new student()
$student->setName('John');
$student->SetLastName('Doe');
$fullName = $student->getName.' '.$student->getLastName();

Answer

You cannot use the $this on a static method since static methods don't have objects associated with them. What would work perfectly would be using the 'paamayim nekudotiam' to call these methods, but then again, a static method would have to be something un-dynamic (as it's called: static) such as a constant, even date/time (even though time changes, it would always show "a time" and not any other inter-changeable or conditional event driven result, it would stay as a time result.

Judging by the nature of this class, it is supposed to be dynamic, hence I would remove the static keyword from both methods and happily use $this to call them from the object instantiated.
